# C-Project-Mini-Git
Мини-Git: персистентное хранилище версий текстовых файлов Цель: на практике понять, как работает персистентность (неизменяемость версий), структурное разделение (structural sharing), хеширование содержимого и цепочка коммитов — всё то, что лежит в основе настоящего Git.
# Этап 1: Базовая архитектура и инициализация
	Инициализация репозитория `init_repo`: Функция должна создавать пустой репозиторий. Она возвращает начальный пустой коммит.
# Этап 2: Управление файлами
	Добавление и обновление файла `add_file`: Функция принимает текущий коммит, путь к файлу и его содержимое. Возвращает новый коммит, в котором этот файл добавлен или обновлен. Старый коммит должен оставаться без изменений.
	Удаление файла `remove_file`: Функция принимает текущий коммит и путь к файлу. Возвращает новый коммит, в котором указанный файл удален.
# Этап 3: Фиксация изменений и чтение данных
	Фиксация коммита `commit`: Функция принимает текущее промежуточное состояние и сообщение коммита. Она создает финальный коммит с вычисленным хешем и привязкой к родителю. Возвращает новый коммит.
	Чтение содержимого `get_file_content`: Принимает коммит и путь к файлу. Возвращает содержимое файла в виде строки или NULL, если файла не существует.
	Проверка наличия файла `get_file_exists`: Принимает коммит и путь. Возвращает true или false в зависимости от того, есть ли файл в текущей версии.
# Этап 4: Информационные утилиты и логирование
	Информация о коммите `print_commit`: Выводит данные одного коммита: хеш, родительский хеш, сообщение, дату и список измененных файлов.
	История коммитов `print_history`: Принимает коммит и выводит всю цепочку истории назад до начального коммита.
	Просмотр дерева файлов `print_files`: Принимает коммит и выводит список всех файлов вместе с их хешами в этой версии.
# Этап 5: Дополнительный функционал (Ветвление и слияние)
	Создание ветки `create_branch`: Принимает коммит и имя ветки. Запоминает коммит как начало новой ветки.
	Получение головы ветки `get_branch_head`: Принимает имя ветки. Возвращает последний коммит указанной ветки.
	Переключение `checkout`: Принимает коммит и делает его текущим.
	Простое слияние `merge_simple`: Принимает два коммита. Создает новый коммит, который содержит файлы из второго коммита поверх первого, без разрешения конфликтов.
# Этап 6: Оптимизация и статистика
	Подсчет объектов `count_objects`: Принимает коммит. Подсчитывает, сколько уникальных blob и tree объектов существует во всей истории от этого коммита.
	Анализ экономии памяти `show_memory_saving`: Выводит статистику о количестве скопированных узлов при последней операции по сравнению с полным копированием дерева.
