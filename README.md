# C-Project-Mini-Git

> Студентам предлагается реализовать на языке C упрощённую персистентную систему
контроля версий файлов, работающую по принципам, аналогичным Git. Программа должна
позволять создавать последовательность неизменяемых состояний (коммитов), в каждом из
которых хранится набор текстовых файлов; при добавлении, изменении или удалении файла
создаётся новая версия репозитория, при этом неизменённые файлы не копируются заново,
а используются по ссылке (структурное разделение, structural sharing). Реализация и
сравнительный анализ приближённых алгоритмов для обобщённой задачи
распределения нагрузки с ограничениями на машины.

## Быстрый старт

Компиляция
```bash
make
```
Запуск
```bash
./minigit
```
Пример работы
```bash
./minigit init
echo "Hello, World!" > file1.txt
echo "This is a test file." > file2.txt
./minigit add file1.txt
./minigit commit "Добавлен файл file1.txt"
./minigit add file2.txt
./minigit commit "Добавлен файл file2.txt"
./minigit log
```

### Цель
- *На практике понять, как работает персистентность (неизменяемость версий), структурное разделение `structural sharing`, хеширование содержимого и цепочка коммитов — всё то, что лежит в основе настоящего <a href=https://git-scm.com/>Git</a>*.

- **Основная идея** - Продемонстрировать персистентность данных: любая предыдущая версия
остаётся полностью доступной и неизменной даже после множества последующих
коммитов. В рамках проекта требуется реализовать базовые операции: инициализацию
репозитория, добавление/изменение/удаление файла в новой версии, фиксацию коммита с
сообщением, получение содержимого любого файла из любой версии истории, а также
вывод списка файлов и цепочки коммитов. Дополнительно приветствуется поддержка
ветвления, простого слияния и подсчёта экономии памяти за счёт совместного
использования неизменённых частей структуры.

### Этап 3: Фиксация изменений и чтение данных
- **Фиксация коммита** (`commit`): Функция принимает текущее промежуточное состояние и сообщение коммита. Она создает финальный коммит с вычисленным хешем и привязкой к родителю. Возвращает новый коммит.
- **Чтение содержимого** (`get_file_content`): Принимает коммит и путь к файлу. Возвращает содержимое файла в виде строки или NULL, если файла не существует.
- **Проверка наличия файла** (`get_file_exists`): Принимает коммит и путь. Возвращает true или false в зависимости от того, есть ли файл в текущей версии.
